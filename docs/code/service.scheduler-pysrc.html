  1  ''' 
  2  This is the application scheduler.  
  3  It defines scheduled tasks and runs them as per their defined schedule. 
  4   
  5  This scheduler is started and stopped when the app is started and stopped.  
  6  It is presumed to run on one machine at present. 
  7   
  8  If scaling later requires having multiple machines, then this scheduler should only run on the machine that has access to  
  9  the relevant directories. There is a task that moves files from ftp user jail directories to tmp processing locations, and  
 10  this is the limitation - creating sftp accounts has to happen on one machine or across machines, but that would increase  
 11  attack surface for security vulnerability. So probably better to have only one machine open to sftp, and if necessary for  
 12  later scale the script that is called to move data from the sftp jails to processing locations could do so by round-robin  
 13  to multiple processing machines. The jper app config has settings for running this scheduler and what frequencies to run each  
 14  process, so it is just a case of installing jper on each machine but only setting the frequencies for the processes desired to  
 15  be scheduled on each given machine. 
 16   
 17  Or, if scheduled tasks themselves also need to be scaled up, the scheduler can continue to run on  
 18  all machines but some synchronisation would have to be added to that tasks were not run on every machine. Also, each machine  
 19  running the schedule would need access to any relevant directories. 
 20  '''
