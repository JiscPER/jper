Technical documentation

Jisc Publications Router is a messaging system that receives notifications from publishers and other sources (e.g. Europe PMC) about article publications and routes these using a matching algorithm to particular Institutional Repositories. At its simplest, a notification is simply a metadata package providing details of an article that has been either accepted for publication or has been published. The notification may also include the article text either embedded in the XML or as one or more separate binary files, for example a PDF or WORD document and graphics files.

Publishers can provide Publications Router with notification metadata in one of two formats, either JATS XML or the Router's proprietary API JSON structure, via any of these mechanisms:

  1. FTP: sending JATS XML metadata plus any binary files as a zipped package (or unzipped if particular rules are followed)
  2. SWORD2 deposit: as per SWORD2 specifiction
  3. Publication Router API: a REST web service (Publication Router's JSON structure).

Institutions may receive the notifications directly into their Repository or CRIS (if the vendor supports this) by any of these mechanisms:

  1. SWORD2 deposit: which allows Publication Router to push the notifications to those repositories which support this protocol (such as Eprints and DSpace)
  2. OAI-PMG: which enables repositories to pull the notficitions from Publication Router via an exposted end-point (URL)
  3. Publication Router API: which enables repositories or CRISs to pull notifications via a REST web service.

In order to use Publications Router both Publishers and Institutions must have completed the sign-up process which results in creation of a Publication Router Account which is used to configure and administer Publication Router settings that are necessary for inter-connection.
